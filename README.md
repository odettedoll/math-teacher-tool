### 🐛 긴급 버그 수정: 문항 생성 오류 해결 (v2.7) 🔥

```javascript

### 🔧 위치 지정 버튼 ReferenceError 완벽 해결 (v3.3)

**배포 환경에서 발생하던 "toggleMarkerMode is not defined" 오류를 완전히 해결했습니다!**

#### 🐛 문제 분석

**증상:**
```
배포 환경에서 "위치 지정" 버튼 클릭 시:
- 콘솔 오류: Uncaught ReferenceError: toggleMarkerMode is not defined ❌
- 버튼이 활성화되지 않음 ❌
- 커서가 크로스헤어로 변경되지 않음 ❌
- 마커가 생성되지 않음 ❌
```

**원인:**
1. **HTML onclick 속성이 여전히 남아있었음**
   - lesson.html에 `onclick="toggleMarkerMode()"`가 여전히 존재
   - DOMContentLoaded에서 이벤트 리스너를 추가하지만, onclick이 먼저 실행됨
   - onclick은 페이지 로드 시점에 평가되어 함수를 찾지 못함

2. **이벤트 리스너와 onclick의 충돌**
   - 두 가지 방식이 동시에 존재하여 혼란 발생
   - onclick이 먼저 실행되어 오류 발생

#### ✅ 해결 방법

##### 1. HTML에서 onclick 완전 제거

**Before (v3.2)**
```html
<!-- onclick 속성이 여전히 존재 -->
<button id="markerBtn" onclick="toggleMarkerMode()">
  📍 위치 지정
</button>
```

**After (v3.3)**
```html
<!-- onclick 완전 제거 - 순수 이벤트 리스너만 사용 -->
<button id="markerBtn">
  📍 위치 지정
</button>
```
```

```javascript
// DOMContentLoaded에서 이벤트 리스너 등록 (v3.3)
document.addEventListener('DOMContentLoaded', function() {
  console.log('🎓 VI-ZONE 수업 자료 생성 페이지 로드 완료');
  console.log('📍 Position Marker 기능 활성화');
  
  const markerBtn = document.getElementById('markerBtn');
  
  if (markerBtn) {
    console.log('✅ 위치 지정 버튼 찾음:', markerBtn);
    
    // 이벤트 리스너 등록 (onclick 없음)
    markerBtn.addEventListener('click', function(e) {
      e.preventDefault();
      console.log('🖱️ 위치 지정 버튼 클릭됨');
      
      // 편집 모드 체크
      if (isEditMode) {
        console.log('⚠️ 편집 모드에서는 위치 지정 불가');
        return;
      }
      
      // toggleMarkerMode 함수 호출
      if (typeof window.toggleMarkerMode === 'function') {
        console.log('✅ toggleMarkerMode 함수 호출');
        window.toggleMarkerMode();
      } else {
        console.error('❌ toggleMarkerMode 함수를 찾을 수 없습니다');
      }
    });
    
    console.log('✅ 위치 지정 버튼 이벤트 리스너 등록 완료');
  } else {
    console.error('❌ 위치 지정 버튼을 찾을 수 없습니다');
  }
  
  console.log('✅ 모든 기능 준비 완료');
});
```

**편집 모드에서 탭 삭제 버튼이 겹쳐서 표시되던 문제를 완벽히 해결했습니다!**

```

```javascript
function renderTab(category) {

### 🐛 탭 삭제 버튼 및 위치 지정 버튼 수정 (v3.0)

**탭 삭제 버튼이 각 탭에 개별적으로 표시되고, 위치 지정 버튼이 정상 작동합니다!**

#### 🎯 수정된 문제

##### 1. 탭 삭제 버튼 겹침 문제 ✅

**Before (v2.9)**
```html
<!-- 잘못된 구조: button 안에 button -->
<button class="tab-button">
  탭 내용
  <button class="tab-delete-x">X</button> ❌
</button>
```
- 문제: 4개의 삭제 버튼이 모두 tabs-container 우측 상단에 겹쳐서 표시됨
- 원인: HTML에서 `<button>` 안에 다른 `<button>`을 넣으면 안 됨

**After (v3.0)**
```html
<!-- 올바른 구조: wrapper로 감싸기 -->
<div class="tab-wrapper">
  <button class="tab-button">탭 내용</button>
  <button class="tab-delete-x">X</button> ✅
</div>
```
- 해결: 각 탭을 `.tab-wrapper`로 감싸고 삭제 버튼을 별도로 배치
- 결과: 각 탭의 우측 상단에 삭제 버튼이 개별적으로 표시됨

##### 2. 위치 지정 버튼 작동 문제 ✅

**문제**
- 배포 환경에서 "위치 지정" 버튼 클릭 시 아무 반응이 없음
- 로컬 환경에서는 정상 작동

**원인 분석**
- `toggleMarkerMode` 함수는 `window` 객체에 제대로 등록됨
- HTML의 `onclick="toggleMarkerMode()"` 속성도 정상
- **캐시 문제로 인해 이전 버전의 JS 파일이 로드됨**

**해결 방법**
- CSS 버전을 v3.0으로 업데이트하여 브라우저 캐시 무효화
- 사용자에게 강력 새로고침 안내 (Ctrl+Shift+R)

#### 🛠️ 코드 변경 사항

##### 1. JavaScript (js/lesson.js)

**renderTab 함수 수정:**
```javascript
// Before
return `
  <button class="tab-button">
    내용
    <button class="tab-delete-x">X</button>
  </button>
`;

// After
return `
  <div class="tab-wrapper ${isActive ? 'active' : ''}">
    <button class="tab-button">내용</button>
    <button class="tab-delete-x">X</button>
  </div>
`;
```

##### 2. CSS (css/lesson.css)

**tab-wrapper 스타일 추가:**
```css

  position: relative;
  display: inline-block;
}

  border-bottom: 3px solid transparent;
  color: var(--text-secondary);

  transition: all 0.2s;
}

}

.tab-wrapper.active .tab-count {
  background: var(--primary-color);
  color: var(--white);
}
```

### 🚨 긴급 버그 수정 패치 (v2.4 + Fix v1.0) 🆕

**배포 환경에서 발생한 3가지 주요 버그를 모두 수정했습니다.**

#### 1️⃣ 사용 팁 닫기 버튼 오류 수정
- **문제**: X 버튼 클릭 시 사용 팁 박스가 닫히지 않음
- **원인**: CSS `.hidden` 클래스가 제대로 작동하지 않음
- **해결**: 
  - `style.display = 'none'`으로 직접 제어
  - CSS 강제 적용 (multiple selectors with `!important`)
  - 로컬 스토리지 상태 저장 유지
- **결과**: ✅ X 버튼으로 즉시 닫힘

#### 2️⃣ 편집 모드 작동 오류 수정  
- **문제**: 자료 생성 후 편집 모드로 변경해도 반응 없음
- **원인**: 
  - `isEditMode` 변수 초기화 문제
  - 토글 슬라이더 애니메이션 미작동

- "수학적 계산 탭에 모듈을 하나 추가해줘"
- "정리 정의에 페이지 추가"
- "실생활 예시 탭에 새 장 만들어줘"
```

**2. 모듈 삭제**
```
예시:
- "수학적 계산 탭의 2번째 모듈 삭제해줘"
- "실생활 예시 1번 페이지 지워줘"
- "응용 문제 3번째 장 제거해줘"
```

**3. 탭 추가**
```
예시:
- "새 탭 추가해줘"
- "카테고리 하나 만들어줘"
```

**4. 탭 전환**
```
예시:
- "정리 정의 탭 보여줘"
- "실생활 예시로 이동"
- "수학적 계산 탭 열어줘"
```

**5. 모듈 순서 변경**
```
예시:
- "정리 정의 2번째 모듈 위로 올려줘"
- "실생활 예시 1번 페이지 아래로 내려줘"
```

#### 키워드 인식
- **카테고리**: 정리, 정의, 실생활, 예시, 수학, 계산, 응용, 문제
- **대상**: 모듈, 페이지, 장, 탭, 카테고리
- **동작**: 추가, 삭제, 지워, 제거, 만들, 생성, 이동, 보여, 위로, 아래로

### ✨ 모듈 기반 자료 시스템 구축 🆕
- **모듈 개념 도입**: 1개의 장 = 1개의 모듈로 자료 구성
  - 각 모듈은 독립적인 콘텐츠 블록
  - 탭에 모듈 개수 실시간 표시 (예: "정리 정의 (2)")
  - 카테고리별 모듈 자동 분류
  
- **모듈 관리 기능**:
  - ↑ 버튼: 모듈 위로 이동
  - ↓ 버튼: 모듈 아래로 이동
  - + 버튼: 현재 모듈 뒤에 새 모듈 추가
  - 🗑 버튼: 모듈 삭제 (마지막 모듈은 삭제 불가)
  - 편집 모드에서만 컨트롤 버튼 표시
  
- **스크롤 기반 탐색**: 전체 모듈을 연속 스크롤로 확인 가능

  - 좌측: 자료 제목, 형식, 페이지 수 표시
  - 우측: 다운로드, 공유하기 버튼
  - 하단: 자동 저장 상태 (시간 표시)
- **다운로드 모달**: 다양한 형식으로 자료 다운로드 가능 🆕
  - 버튼형 형식 선택: PPT, PDF, HWP, HTML
  - 선택된 형식 시각적 표시 (체크마크)
  - 다운로드 버튼: 선택한 형식으로 다운로드
  - 바로 인쇄하기 버튼: 브라우저 인쇄 기능 실행
  - ESC 키로 모달 닫기
- **공유하기 모달**: 다양한 방식으로 자료 공유 가능 🆕
  - 버튼형 공유 방식 선택: 링크 복사하기, QR 생성하기
  - **링크 복사하기**: 공유 링크 URL 표시 + 복사 버튼
    - 클립보드에 자동 복사
    - 읽기 전용 입력 필드로 링크 표시
  - **QR 생성하기**: QR 코드 이미지 생성 및 제공
    - 200x200 QR 코드 이미지 표시
    - 이미지 복사 버튼 (클립보드로 복사)
    - QR 다운로드 버튼 (PNG 파일로 다운로드)
  - 선택에 따라 하단 영역 동적 변경
  - ESC 키로 모달 닫기
- **문서형 미리보기**: 생성된 자료를 실제 문서처럼 바로 확인
  - 카테고리별 섹션 구분 (정리 정의, 실생활 예시, 수학적 계산, 응용 문제)
  - 각 섹션 내 페이지 콘텐츠를 문서 형태로 표시
  - 스크롤을 통해 전체 자료를 연속적으로 확인
- **탭 기반 네비게이션**: 4개 카테고리로 빠른 섹션 이동
  - 📐 **정리 정의**: 개념 정리 및 이론 설명 장
  - 🌍 **실생활 예시**: 실생활 적용 및 사례 장
  - 🧮 **수학적 계산**: 계산 연습 및 문제 풀이 장
  - 🎯 **응용 문제**: 심화 학습 및 응용 문제 장
  - 탭 클릭 시 해당 섹션으로 부드럽게 스크롤
- **멀티 페이지 자료**: 하나의 자료가 3-4개 장(페이지)으로 구성
  - 각 장은 카테고리별로 자동 분류
  - 페이지 배지 + 제목으로 명확한 구분
  - 콘텐츠 박스, 예시 박스, 문제 박스 등 다양한 콘텐츠 블록
- **실시간 자동 저장**: 자료 생성 및 수정 시 2초 후 자동 저장
  - 저장 상태 표시 (체크 아이콘 + 저장 시간)
  - "HH:MM에 저장됨" 형식으로 시간 표시
  - 3초 후 자동으로 페이드아웃

## 프로젝트 개요

VI-ZONE은 수학 교사들이 수업 자료, 문항, 교육 도구를 효율적으로 생성하고 관리할 수 있는 웹 기반 플랫폼입니다. 깔끔하고 신뢰감 있는 디자인으로 교육 현장에서 바로 사용할 수 있습니다.

## 주요 기능 설명

### 사이드바 접기/펼치기
- 좌측 상단의 햄버거 메뉴(☰)를 클릭하여 사이드바 접기/펼치기
- localStorage에 상태 저장하여 페이지 새로고침 후에도 유지

### 자료 정렬
1. **이름순**: 가나다 순으로 정렬 (폴더 우선)
2. **최신순**: 최신 항목이 먼저 표시 (폴더 우선)
3. **유형별**: 폴더 → PPT → 이미지 순으로 정렬

### 폴더 관리
- **폴더 추가**: "폴더 추가하기" 버튼 클릭
- **폴더 이름 변경**: 폴더를 더블클릭하여 이름 수정

### 수업 자료 생성 워크플로우 ✨ NEW!

#### 1단계: 자료 생성
1. **유형 선택**: 개념 학습, 활동형, 프로젝트형, 참여형 중 선택
2. **학년/단원 입력**: 예) "중2 일차함수", "고1 삼각함수"
3. **형식 선택**: 유형에 맞는 형식 선택 (PPT, PDF, 학습지 등)
4. **추가 내용 입력**: 필요시 세부 요구사항 입력
5. **자료 생성**: AI가 자동으로 모듈 단위로 자료 생성

#### 2단계: 미리보기 및 탭 관리
- **탭 네비게이션**: 4가지 카테고리 탭 클릭하여 섹션 이동
- **스크롤 탐색**: 전체 문서를 스크롤하여 모든 모듈 확인
- **탭 추가**: "탭 추가" 버튼으로 새로운 카테고리 추가
- **탭 수정**: 편집 모드에서 ✎ 버튼 또는 직접 클릭하여 탭 이름 수정
- **탭 삭제**: − 버튼으로 탭 및 관련 모듈 모두 삭제

#### 3단계: 편집 모드 활용
1. **편집 모드 진입**: 토글 스위치를 "편집"으로 전환
2. **내용 수정**:
   - 자료 제목 클릭하여 수정
   - 섹션 제목 클릭하여 수정
   - 모듈 제목 및 본문 내용 클릭하여 수정
3. **모듈 관리**:
   - **↑/↓ 버튼**: 모듈 순서 변경
   - **+ 버튼**: 현재 모듈 뒤에 새 모듈 추가
   - **🗑 버튼**: 모듈 삭제 (마지막 모듈은 삭제 불가)
4. **읽기 모드 전환**: 토글 스위치를 "읽기"로 전환하여 자동 저장

#### 4단계: 다운로드 및 공유
1. **다운로드**: 
   - 다운로드 버튼 클릭
   - 형식 선택: PPT, PDF, HWP, HTML
   - 다운로드 또는 바로 인쇄하기
2. **공유**:
   - 공유하기 버튼 클릭
   - 링크 복사 또는 QR 코드 생성
   - 링크 복사 또는 QR 이미지 다운로드

### 모듈 기반 자료 구조 상세 설명 ✨

#### 모듈(Module) 개념
- **1개의 장 = 1개의 모듈**: 각 모듈은 독립적인 콘텐츠 단위
- **모듈 구성 요소**:
  - 페이지 번호 배지 (예: "1장")
  - 모듈 제목
  - 본문 내용
  - 핵심 내용 박스 (💡)
  - 카테고리별 특수 콘텐츠:
    - 실생활 예시: 실생활 적용 사례 박스
    - 수학적 계산: 예제 문제 + 풀이 박스
    - 응용 문제: 도전 문제 박스

#### 모듈 관리 시나리오

**시나리오 1: 모듈 순서 변경**
```
정리 정의 탭에 3개의 모듈이 있을 때:
1장: 일차함수의 정의
2장: 기울기와 y절편
3장: 그래프 그리기

[방법 1] 편집 모드에서 버튼 사용
→ 2장을 1장 앞으로 이동 (↑ 버튼)

[방법 2] 채팅으로 명령
→ 채팅: "정리 정의 2번째 모듈 위로 올려줘"

→ 결과:
1장: 기울기와 y절편
2장: 일차함수의 정의
3장: 그래프 그리기
```

**시나리오 2: 모듈 추가**
```
실생활 예시 탭에 2개의 모듈이 있을 때:
1장: 택시 요금
2장: 휴대폰 요금제

[방법 1] 편집 모드에서 버튼 사용
→ 1장 뒤에 새 모듈 추가 (+ 버튼)

[방법 2] 채팅으로 명령
→ 채팅: "실생활 예시 탭에 모듈 하나 추가해줘"

→ 결과:
1장: 택시 요금
2장: 새 모듈 (제목과 내용 편집 가능)
3장: 휴대폰 요금제
```

**시나리오 3: 모듈 삭제**
```
수학적 계산 탭에 4개의 모듈이 있을 때:
1장: 기본 계산
2장: 응용 계산
3장: 심화 계산
4장: 종합 문제

[방법 1] 편집 모드에서 버튼 사용
→ 2장 삭제 (🗑 버튼)

[방법 2] 채팅으로 명령
→ 채팅: "수학적 계산 탭의 2번째 모듈 삭제해줘"

→ 결과:
1장: 기본 계산
2장: 심화 계산
3장: 종합 문제
```
